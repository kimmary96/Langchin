## 1. 서비스 기획

### 서비스 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | 어디가 아프니? (엄마품) |
| **한 줄 정의** | 내 병력을 다 알고 있는 엄마 같은 친구가 아플 때 곁에서 위로해주고 건강을 함께 관리해주는 AI 헬스케어 챗봇 |
| **핵심 가치** | 아플 때 혼자가 아닌 느낌 — 심리적 위로 + 실질적 건강 정보 제공 |

### 타겟 사용자

- **주요 타겟**: 만성 질환이나 반복 증상을 가진 30~50대 여성

- **문제 상황**:

| # | 문제 상황 |
|---|-----------|
| 1 | 아프다고 말해도 공감받지 못하는 서운함 |
| 2 | 반복 증상을 주변에 털어놓기 미안함 |
| 3 | 내 병력을 모르는 챗봇에게 매번 처음부터 설명해야 하는 피로감 |
| 4 | 언제 어떻게 아팠는지 기록이 없어 병원 설명이 어려움 |

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 엄마 페르소나 챗봇 | LangGraph 기반 Plan → Execute → Refine 3단계 그래프로 공감 + 정확성을 갖춘 답변 생성 |
| 병력 등록 및 관리 | 사이드바에서 직접 등록 + 대화 중 자동 추가 (질환명·날짜·메모 저장) |
| 건강일기 캘린더 | 컨디션·증상·배변·수면·운동·병원·메모 7개 카테고리 기록 |
| 약봉투/상처 사진 분석 | OpenAI Vision API(GPT-4o)로 약 이름·효능·용법 또는 상처 상태 분석 |
| 증상 기반 병원 추천 | 60개+ 증상 키워드 → 진료과 매핑 후 네이버 지도 연동 |
| 오늘의 건강 리포트 | 하루 대화 요약을 자동 생성하여 건강 흐름 파악 가능 |

### 한계점 및 개선 방향

**한계점**

| 항목 | 설명 |
|------|------|
| 의학적 진단 불가 | 법적 한계로 인해 진단·처방 기능 제공 불가 |
| 임상 검증 미실시 | 실제 환자 대상 임상 검증을 거치지 않음 |
| 단일 사용자 | 로그인/인증 없이 1인 사용 전제 |
| 복약 알림 미구현 | 알림 UI는 존재하나 실제 백그라운드 발송 미구현 |

**개선 방향**

| 시기 | 개선 내용 |
|------|-----------|
| 단기 | 응답 스트리밍, RAG 기반 의학 정보 정확도 향상 |
| 중기 | 증상 위험도 자동 분류, 월간 건강 트렌드 분석 |
| 장기 | 다중 에이전트, 가족 공유 기능, FastAPI 전환 |

## 2. 기술 구조

### 기술 스택

| 분류 | 기술 | 설명 |
|------|------|------|
| AI Agent | `langgraph` | Plan → Execute → Refine 3단계 그래프 구조 |
| LLM | `langchain-openai` (GPT-4o-mini) | 대화 생성·계획·검토에 사용 |
| 웹 검색 | `tavily-python` | 의학 정보·약 정보 실시간 검색 |
| Vision | `openai` (GPT-4o) | 약봉투·상처 사진 분석 |
| UI | `streamlit` | 모바일 퍼스트 챗봇 인터페이스 |
| 프롬프트 관리 | `pyyaml` | 시스템 프롬프트 YAML 구조화 |
| 데이터 저장 | JSON 파일 (로컬) | 병력·건강일기·대화 이력 저장 |

### LangGraph 구조 설명

본 서비스는 **Plan → Execute → Refine** 3단계 그래프 구조로 동작합니다.

**1단계: Plan 노드 (계획) — temperature: 0.3**

사용자 메시지와 등록된 병력을 분석하여 답변 전략을 수립합니다. 이 단계에서 웹 검색 필요 여부(`search_needed`)를 판단하며, 약 이름·특정 질병명·최신 의학 정보가 포함된 질문은 `search_needed=True`를 강제합니다. 낮은 temperature(0.3)로 정확하고 일관된 계획을 세웁니다.

**2단계: Execute 노드 (실행) — temperature: 0.8**

Plan 단계의 전략에 따라 엄마 페르소나로 답변을 생성합니다. `search_needed=True`인 경우 Tavily 웹 검색을 수행한 후 결과를 반영하여 답변합니다. 재시도 시에는 Refine 노드의 피드백을 반영하여 개선된 답변을 생성합니다. 높은 temperature(0.8)로 자연스럽고 따뜻한 대화 톤을 구현합니다.

**3단계: Refine 노드 (검토 + Cycle) — temperature: 0.1**

생성된 답변의 품질을 공감·정확성·안전성 기준으로 평가합니다. 평가 점수가 7점 미만이면 Execute 노드로 되돌아가는 **Cycle**을 구현하여 답변 품질을 보장합니다. 최대 재시도 1회(`MAX_RETRIES=2`)로 무한 루프를 방지하며, 매우 낮은 temperature(0.1)로 엄격한 평가를 수행합니다.

**조건부 분기:**

- `after_plan`: `search_needed=False` → `execute_direct` (Refine 없이 바로 종료)
- `after_plan`: `search_needed=True` → `execute` → `refine` → `should_retry`
- 이를 통해 **일반 대화는 LLM 2회 호출**, **검색 대화는 최대 5회 호출**로 효율화

## 3. 사용 시나리오

### 시나리오 1: 일반 증상 공감 (검색 없이)

사용자가 "오늘 머리가 너무 아파"라고 말하면, 엄마품은 즉각적인 공감 반응을 보입니다. Plan 노드에서 웹 검색이 필요하지 않다고 판단(`search_needed=False`)하고, Execute 노드에서 바로 엄마 페르소나로 따뜻한 위로와 함께 등록된 병력을 연계한 맞춤 조언을 전달합니다. Refine 단계를 건너뛰고 **LLM 2회 호출**만으로 빠르게 응답하여 사용자가 기다리는 시간을 최소화합니다. "우리 딸 머리 또 아프구나... 요즘 스트레스 많이 받았지?"처럼 반복 증상에 대한 기억과 공감을 담은 응답을 제공합니다.

### 시나리오 2: 의학 정보 검색

사용자가 "내가 듀파스톤이라는 약 먹고 있는데 뭔지 알아?"라고 물으면, Plan 노드에서 약 이름이 포함된 질문임을 감지하고 `search_needed=True`를 설정합니다. Execute 노드에서 **Tavily 웹 검색**을 실행하여 듀파스톤의 성분·효능·용법에 대한 최신 정보를 수집한 뒤, 검색 결과를 바탕으로 정확한 약 정보를 엄마 말투로 전달합니다. Refine 노드에서 의학 정보의 정확성과 안전성을 검토한 후 최종 응답을 제공합니다. 검색 기반 응답은 **초록 테두리 말풍선**으로 표시되어 사용자가 일반 대화와 구분할 수 있습니다.

### 시나리오 3: 대화 중 병력 자동 추가

사용자가 대화 중 "나 류마티스 관절염 있어, 병력에 추가해줘"라고 말하면, 엄마품은 먼저 "아이고, 관절이 많이 아프겠다..."와 같은 공감 반응을 보낸 뒤, **"병력에 추가할까?"** 확인 버튼을 표시합니다. 사용자가 확인을 누르면 류마티스 관절염이 병력 목록(`medical_history.json`)에 저장되고, 이후 모든 대화에서 이 정보가 자동으로 시스템 프롬프트에 반영됩니다. 다음에 "오늘 손가락이 뻣뻣해"라고 말하면, 엄마품은 등록된 류마티스 관절염 병력을 참고하여 "류마티스 때문에 그런 거 아닐까? 오늘 날씨도 습하더라..."와 같이 병력 맥락을 반영한 맞춤 응답을 제공합니다.
